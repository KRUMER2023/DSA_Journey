# 1915. Number of Wonderful Substrings

## Intuition
A substring is called wonderful if it contains at most one character whose frequency is odd.

Instead of checking every substring one by one, we use prefix parity and a bitmask.

For each prefix of the string, we keep track of whether the count of each character is odd or even. A compact way to store this is with a bitmask:
- bit `1` means that character count is odd
- bit `0` means that character count is even

If two prefixes have the same mask, then the substring between them has all character counts even, which is wonderful.

If two prefixes differ by exactly one bit, then the substring between them has exactly one odd character count, which is also wonderful.

## How the Code Works
1. Start with an empty prefix mask `0` and store `mp = {0: 1}`.
   - This means: before processing any characters, there is one way to have the empty prefix.
2. Traverse the string from left to right.
3. Update the current mask by toggling the bit for the current character.
4. Check if the current mask was seen before.
   - If yes, the substring ending at the current position is wonderful.
5. Also check all possible one-bit flips of the current mask.
   - If one of those masks was seen before, then the substring between those two prefixes has exactly one odd count.
6. Record the current mask in the frequency map.

## Why This Is Efficient
The key idea is that we never compare substrings directly. We only compare prefix states.

- Each position is processed once.
- For each position, we try a small number of bit flips.
- This turns the solution into an efficient linear-time approach.

## Time Complexity
- O(n * 10) in this implementation, because we check a small set of bit-flip possibilities for each position.
- In big-O notation, this is effectively O(n).

## Space Complexity
- O(number of unique masks), which is small and effectively O(1) for this problem.
